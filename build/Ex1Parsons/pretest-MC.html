Pretest Multiple-Choice Questions¶

Please answer the following multiple-choice questions to the best of your ability. Click the button when you are ready to begin the exam. Click on the button below the question to display the next question. Click on the button when you have answered all the questions that you can answer.

      2-1-1: What does the following function return from mystery([2, 5, 8, 3, -1],1,3)?

      def mystery(numList, start, end):
    save = numList[start]
    for index in range(start,end+1):
       value = numList[index]
       if value < save:
           save = value
    return save
    • (A) -1
    • This answer would be true if the end value was 4 rather than 3.
    • (B) 2
    • This answer would be true if the start value was 0 rather than 1.
    • (C) 3
    • This would be true if it was looking for the largest value in the list from start to end (inclusive), but it finds the smallest value.
    • (D) 5
    • The range function returns a list from first to (last - 1) so this will look for the smallest value from index 1 to 3 and so return 3.
    • (E) 8
    • This would be true if range returned a list of values from the first to last but it returns from first to last - 1.

      2-1-2: Consider the following function findLongest which is intended to find the longest consecutive block of the value target occurring in the list nums; however findLongest does not work as intended. For example, if the list nums contains the values [7, 10, 10, 15, 15, 15, 15, 10, 10, 10, 15, 10] the call findLongest(10, nums) should return 3, the length of the longest consecutive block of 10s. Which of the following best describes the value actually returned by a call to findLongest?

      def findLongest(target, nums):
    lenCount = 0
    maxLen = 0
    for num in nums:
        if num == target:
            lenCount = lenCount + 1
        else:
            maxLen = lenCount
    if lenCount > maxLen:
        maxLen = lenCount
    return maxLen
    • (A) It is the length of the shortest consecutive block of the value target in nums
    • It doesn't reset lenCount ever, so it just counts all the times the target value appears in nums.
    • (B) It is the length of the array nums
    • It only increments the lenCount when the current num is equal to the target.
    • (C) It is the length of the first consecutive block of the value target in nums
    • It doesn't reset lenCount ever, so it just counts all the times the target value appears in the array.
    • (D) It is the number of occurrences of the value target in nums
    • The variable lenCount is incremented each time the current array element is the same value as the target. It is never reset so it counts the number of occurrences of the value target in nums. The method returns maxLen which is set to lenCount after the loop finishes if lenCount is greater than maxLen.
    • (E) It is the length of the last consecutive block of the value target in nums
    • It doesn't reset lenCount ever, so it just counts all the times the target value appears in the array.

      2-1-3: Given the following function definition, what would be returned from mystery(5, 1, 4, [5, 1, 5, 5, 5])?

      def mystery(target, start, end, numList):
    count = 0
    for index in range(start, end):
        current = index
        if current == target:
            count = count + 1
    return count
    • (A) 0
    • While this loops through the indices from 1 to 3 it compares the index to the target and so count remains 0.
    • (B) 1
    • This would be true if it compared the value at the index to the target, but it compares the index to the target.
    • (C) 2
    • This would be true if the range included the last value and the code compared the value at the index to the target.
    • (D) 3
    • This would be true if the start value was 0 and the range included the last value, and the code compared the value at the index to the target.
    • (E) 4
    • This would be true if the start value was 0 and the end value was 6 and the code compared the value at the index to the target.

      2-1-4: What do a and b equal after the following code executes?

      a = 10
b = 3
t = 0
for i in range(1,4):
    t = a;
    a = i + b;
    b = t - i;
    • (A) a = 11 and b = 2
    • This would be true if it was range(1,3).
    • (B) a = 12 and b = 1
    • This would be true if it was range(1,5). Remember that range doesn't include the second value.
    • (C) a = 3 and b = 11
    • Not quite. Check your tracing.
    • (D) a = 8 and b = 5
    • Not quite. Check your tracing.
    • (E) a = 5 and b = 8
    • Good job tracing this!

      2-1-5: Given the following code what will mystery([10,20,30],1,2) return?

      def mystery(numList, start, end):
    sum = 0
    for index in range(start,end+1):
        value = numList[index]
        sum = sum + value
    if (end - start + 1) >= 1:
        return sum / (end - start + 1)
    return 0
    • (A) 15.0
    • This would be true if start was 0 and end was 1.
    • (B) 30.0
    • This would be true if start was 2 and end was 2.
    • (C) 20.0
    • This would be true if start was 0 and end was 3.
    • (D) 25.0
    • This is 20 + 30 = 50 / 2 = 25.0.
    • (E) 0
    • This would be true if end was less than start.
